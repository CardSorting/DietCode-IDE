# Phase Roadmap

## Phase 0: Product contract and scaffold

- Product docs.
- Architecture docs.
- Source layout.
- Build system.
- Pure editor/search tests.
- Knowledge ledger.

## Phase 1A: macOS vertical slice

- Native app window.
- Native menus.
- Welcome screen.
- New file.
- Open file.
- Edit text.
- Save and Save As.
- Dirty state.
- Status bar.

## Phase 1B: Familiar IDE shell

- Activity bar with labels.
- Sidebar empty states.
- Basic tabs.
- Find in file.
- Basic settings: theme, font size, tab size, word wrap.
- Unsaved close/quit confirmation.

## Phase 2: Folder navigation

- Open folder.
- Lazy file tree.
- New file/folder from sidebar.
- Rename.
- Delete with confirmation.
- Reveal in Finder/File Explorer.
- Refresh.
- Command palette.
- Go to line.
- Replace in file.
- Recent files/folders.

## Phase 3: Run and output

- Integrated output panel.
- Run current file.
- Python run support.
- Compile and run simple C++ file.
- Stop running process.
- Problems panel from run/compiler output.
- Search across folder, user-triggered and cancelable.
- Basic project templates.

## Phase 4: Accessibility and recovery

- Accessibility polish.
- Beginner tutorial.
- High contrast theme.
- Keybinding editor.
- Performance diagnostics.
- Crash-safe recovery.
- Autosave option.
- Large file mode.

## Phase 5: Optional power features

- Optional Git panel.
- Optional LSP support.
- Optional plugin system.
- Optional debugger.
- Optional local AI helper, disabled by default.
