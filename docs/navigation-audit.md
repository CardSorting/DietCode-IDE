# Navigation Audit

## Industry patterns to mirror

From VSCode:

- Activity bar.
- Sidebar.
- Tabs.
- Command palette.
- Status bar.
- Bottom panel.
- Welcome screen.

From Sublime Text:

- Fast, quiet startup.
- Text-first editing.
- Minimal visual noise.
- Command palette usefulness.

From Xcode/macOS apps:

- Native menu bar.
- Standard dialogs.
- Standard shortcuts.
- Native accessibility.
- Standard Settings behavior.

From beginner IDEs:

- Obvious Run button.
- Clear output/errors area.
- Discoverable menu actions.
- Plain explanations.

## Screen review checklist

For every screen:

- Can a beginner tell where they are?
- Can they tell what is selected?
- Can they tell what to do next?
- Can they go back?
- Can they recover from mistakes?
- Are actions visible without shortcut knowledge?
- Are commands also reachable from the menu?
- Is jargon avoided?
- Is the app doing expensive work without permission?

## Beginner labels

Beginner Mode uses visible labels:

- Files
- Search
- Run
- Errors
- Settings

Developer Mode may use denser layout but should never remove discoverability entirely.
