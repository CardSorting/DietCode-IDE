# Beginner Onboarding Flow

## Principle

The tutorial is optional, dismissible, restartable, and never a modal prison.

## First-run defaults

- Beginner Mode enabled.
- Comfortable density.
- Activity labels visible.
- Tooltips enabled.
- Visible Save button.
- Visible Run button when a runnable file is open.
- Simplified status bar.
- Friendly empty states.
- No terminal process started automatically.
- No folder search or indexing started automatically.

## Built-in tutorial outline

1. Open a folder.
2. Create a file.
3. Write code.
4. Save.
5. Run.
6. Read output.
7. Fix an error.
8. Use search.
9. Change settings.

## Tutorial behavior

- Appears as a sidebar card or welcome-screen checklist.
- Can be dismissed with a clear close button.
- Can be restarted from Help > Learn DietCode Basics.
- Does not block editing.
- Does not auto-run code.
- Uses plain language.

## Example copy

- “Start by opening a folder or creating a new file.”
- “Your files will appear in Files on the left.”
- “Use Run when you want DietCode to run supported files.”
- “Nothing runs unless you ask.”

## Beginner task success criteria

A first-time user should be able to:

- Identify where files appear.
- Identify how to create a file.
- Identify how to save.
- Identify how to run code.
- Identify where output and errors appear.
- Increase font size.
- Quit without losing work.
